# Exercise 02: Progress Checkpoint  
## McAllen Mobile Park Event Calendar

---

## 1. Current State

The project has progressed from concept to a functional front-end prototype. A structured, multi-page website has been created using HTML and CSS, following the design outlined in the project proposal.

### What Exists and Is Functional
- **Landing Page (index.html)**
  - Fully structured layout with header, navigation, sidebars, and main content
  - Embedded Google Calendar displaying events (**live and functional**)
  - Banner image representing the park
- **About Page**
  - Explains purpose of the site and how to use it
- **Submission Page**
  - Functional HTML form capturing:
    - Submitter name
    - Event title
    - Date and start time
    - Event description
    - Multiple image uploads (feature + additional images)
- **Shared Layout System**
  - Consistent structure across all pages
  - Reusable CSS file with responsive styling
- **Navigation**
  - Clean, minimal navigation between core pages
- **Styling**
  - Mobile-aware layout
  - Large readable typography
  - Simplified interface aligned with target audience needs

### In Progress
- Event-detail (flyer) pages are not yet implemented
- Submission workflow is not connected to Google Forms or backend
- Mobile usability testing is partially complete but not formally validated

### Not Yet Started
- Individual event pages linked from calendar or featured cards
- Full integration of submission workflow (Google Forms or processing system)
- Deployment confirmation on GitHub Pages (if not already live)

---

## 2. Progress Against Plan

The current build aligns with the original proposal in several key areas but diverges in others.

### Completed Milestones
- Mobile-first layout structure (implemented and functional)
- Landing page with embedded Google Calendar (**live and displaying real event data**)
- Core site pages (home, about, submission)
- Navigation system and consistent layout
- Use of HTML and CSS for lightweight front-end

### In Progress
- Mobile optimization and validation
- Submission workflow refinement
- Final usability improvements

### Not Fully Met (Deviations from Proposal)
- **Event-detail pages (digital flyers)**  
  Proposal specified placeholder event pages for Project 01. These have not yet been created.
  
- **Google Forms integration**  
  Proposal specified embedding Google Forms. Current implementation uses a static HTML form.

- **JavaScript usage**  
  Proposal mentioned JavaScript, but current implementation relies primarily on HTML/CSS.

### Status Assessment
The project is **on track structurally**, with **core functionality (calendar display) complete**, but still **behind on full workflow completion** relative to the proposal.

### Changes from Original Plan
- Replaced planned Google Form with a temporary HTML form for faster development
- Consolidated navigation and removed the separate contact page to simplify scope
- Prioritized layout and usability over backend or integration complexity

---

## 3. Blockers and Challenges

### Technical Challenges
- Integrating Google tools (Calendar and Forms) introduces complexity
- Calendar integration is complete, but form integration is not
- No backend currently exists to process submissions

### Scope Challenges
- Event-detail pages require additional structure and linking logic
- Submission workflow requires decisions on handling data (Google Form vs manual process)

### Time Constraints
- Limited time requires prioritizing core functionality over full feature completion
- Additional coursework reduces available development time

### Skill Gaps
- Limited experience with integrating third-party services (Google Forms)
- Need to determine whether JavaScript is required before Project 01

### Usability Risks
- Mobile usability has not been fully tested on real devices
- Calendar iframe sizing and layout responsiveness may require adjustment

### Summary of Key Risk
The main risk is **not delivering the full intended workflow** (calendar → event → submission) before Project 01.
